Contact Component

  • Purpose: To provide a contact section with a form, contact information, and an embedded map.
  • Structure: The component is wrapped in a SectionWrapper and contains two main parts: the contact form and the contact information section.

Contact Form

  • Elements:
    => Name Input: An input field for the user's name.
    => Email Input: An input field for the user's email.
    => Phone Input: An input field for the user's phone number.
    => Info Input: An input field for how the user found the service.
    => Send Button: A button to submit the form.
  • Functionality: Users can fill out the form to get in touch with the service provider.

Contact Information

  • Phone Numbers: Three different phone numbers displayed with icons.
  • Email: An email address displayed with an icon.

Embedded Google Map

  • Map Display: An iframe displaying a Google map centered on a specific location.
  • Styling: The map is styled to be responsive and fit within the section.

Usage

    The Contact component can be used as a standalone section on a landing page or any other page that requires a contact form and information display.

Props

    The component does not accept any props. All necessary elements and content are hardcoded within the component itself.

Considerations

  • Accessibility: Ensure that the form fields and buttons are accessible.
  • Validation: Implement client-side and server-side validation for the form fields.
  • Error Handling: Provide user feedback for successful form submission and errors.
  • Styling: Ensure that the component is responsive and looks good on various screen sizes.
