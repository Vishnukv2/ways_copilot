Landing Page

Page Structure

    The landing page is composed of various sections, each encapsulated in a functional component. The SectionWrapper component is used to wrap the sections and apply common styling.

1. Components


  • Hero: Represents the hero section of the landing page, typically including a prominent message or call to action. <Hero/><SectionWrapper className={cn('w-full')}></SectionWrapper>
  • AboutUs: Provides information about the company or service, including history, mission, and values
    <AboutUs /> {/* <ChooseUs /> */}
  • Categories: Displays various categories relevant to the website's offerings, helping users navigate to specific sections of interest.
    <Categories /> {/* <Products /> */}
  • Inspiration: Offers inspirational content or ideas related to the website's theme, engaging users with creative or motivating material.
    <Inspiration />
  • Working: Explains the workflow or process of the service/product offered, giving users an understanding of how it works.
    <Working />
  • ClientReview: Showcases reviews and testimonials from clients, providing social proof and building trust with potential customers.
    <ClientReview />
  • Newsletter: Encourages users to sign up for a newsletter to receive updates, news, or promotional content from the website.
    <Newsletter />
  • ChatBot: Integrates a chatbot for user interaction, allowing users to ask questions and receive immediate assistance.
    <ChatBot />

2. Optional Components

    These components are included in the code but commented out, indicating they are optional or to be included in the future:

  • ChooseUs: {/* <ChooseUs /> /}
  • Products: {/ <Products /> */}

3. SectionWrapper

    <SectionWrapper className={cn('w-full')}>
  • Purpose: A wrapper component to apply common styles and layout properties to all sections. It uses the cn utility function to conditionally apply classes.
  • Props:
    className (string): Additional classes to be applied to the wrapper.

Utility Functions

    1. cn
  • Purpose: A utility function from @/lib/utils used to conditionally combine class names.
    import { cn } from '@/lib/utils';
